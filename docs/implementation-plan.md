# Private Memos 软件实现计划

## 1. 工程初始化

- 搭建 Electron + React + TypeScript + Vite 桌面项目。
- 预留 Android 端代码结构，建议用 Capacitor 复用 React 前端生成 APK。
- 配置 ESLint、TypeScript、测试框架和基础构建脚本。

## 2. 数据模型与本地存储

- 使用 SQLite 作为桌面端本地数据库。
- 设计核心表：记事、私人笔记、提醒、任务、任务步骤、账本、账目、对象、日程、设置、同步队列。
- 先实现本地 CRUD，所有模块先能稳定离线运行。

## 3. 主界面框架

- 顶部全局搜索栏。
- 左侧固定模块栏：首页、记事、私人笔记、提醒、任务、记账、日程表、搜索、设置、账号。
- 中间列表/日历视图，右侧编辑区。
- 做响应式布局，为 Android 端切换到底部导航和单屏操作打基础。

## 4. 核心模块第一阶段

- 记事：无标题正文、按列表/周/月显示、按日期统计、点击编辑、失焦/回车保存、高亮。
- 私人笔记：标题+正文、列表/周/月、锁定防误改、指定高亮。
- 任务：任务列表、完成状态、步骤、普通/每日/限时任务、过期高亮、复制、右键菜单。
- 提醒：时间、备注、重复、15 字限制、到点弹窗。
- 首页：昨日总结、今日待办、今日日程。

## 5. 核心模块第二阶段

- 记账：多账本、收入/支出、类目、对象、备注、日/周/月/年汇总、饼图、柱状图、列表和日历式显示。
- 日程表：周/月视图、农历显示、聚合提醒/任务/记事/笔记、特殊日程、国历/农历重复。
- 搜索：全局搜索、模块内区域搜索、日期范围搜索、记账类目搜索。
- 设置/账号：字体、UI 配色、版本信息、显示名、密码修改。

## 6. 导出与操作细节

- 右键菜单：编辑、删除、复制、导出。
- 文本/任务/提醒支持双击进入编辑。
- 文档类导出 DOCX，账本导出 Excel。
- 复制按钮创建副本并进入编辑状态。

## 7. 局域网同步

- 桌面端作为局域网服务器，提供同步 API。
- Android 端同 Wi-Fi 自动发现或手动检查连接。
- 手机离线或连接失败时写入本地队列，重连后补同步。
- 切换 Wi-Fi 时触发连接检查。
- 同步冲突先采用保守策略：按更新时间覆盖，并记录冲突日志，后续可加手动合并。

## 8. Android APK

- 使用同一套 React 业务界面适配移动端。
- 用 Capacitor 打包 Android 9+ APK。
- 手机端本地存储和同步队列独立实现，保持离线可用。

## 9. 测试计划

- 单元测试：数据模型、任务状态、提醒重复规则、搜索过滤、账本汇总。
- 组件测试：各模块列表、编辑、锁定、高亮、完成勾选。
- Electron 集成测试：窗口启动、数据库读写、提醒弹窗、右键菜单、导出文件。
- 端到端测试：新增/编辑/删除/复制核心数据，切换列表/周/月视图。
- 同步测试：电脑端服务启动、手机端连接、断网入队、重连补传、重复同步幂等。
- 构建测试：Windows 10 可运行包、Android APK 构建通过。

## 建议执行顺序

先做桌面端 MVP：工程骨架、数据库、主布局、记事、私人笔记、任务、提醒、首页。
这部分完成并测试通过后，再做记账、日程、搜索、导出、设置。最后做 Android APK 和局域网同步。
